# `sources/` 论文集 README

## 1. 论文集定位

`project_1_llm_state_machine_modeling/sources/` 是 `project_1` 下专门面向“控制系统状态机自动生成”任务建设的论文集工作区。它不是泛泛的相关工作收藏夹，而是一个为后续状态机自然语言需求/描述数据集服务的样本库。

本论文集在整个博士研究中的角色，是从真实控制系统论文中沉淀可追溯的控制逻辑描述、模式切换叙述、阶段推进规则、故障恢复链路和门控条件，为后续 LLM 建模、验证和修复研究提供原始语料与案例来源。

## 2. 设立宗旨与期望收获

单独建立本论文集，目的是把“可用于状态机建模的数据源”与一般 baseline、综述资料、方法背景文献分开管理。这里希望持续沉淀的不是宽泛知识，而是以下几类可复用产物：

1. 真实控制对象的论文原文 PDF、完整 BibTeX 与可检索文本。
2. 基于原文证据整理出的 `STM.md`，用于沉淀可追溯的状态机自然语言描述。
3. 跨论文的领域分布、检索关键词簇、失败历史和收获盘点，用于持续改进检索和筛选方向。

## 3. 收录范围

本论文集优先收录以下论文：

1. 明确面向具体控制对象、控制器或控制子系统，如电梯、交通灯、铁路联锁、道口门控、起搏器、机械呼吸机、起落架、UAV 任务控制、PLC 离散制造控制等。
2. 文中存在运行模式、状态切换、阶段推进、故障处理、恢复路径、时序约束、守卫条件或门控逻辑，可整理为状态机自然语言描述。
3. 可获得 PDF 原文，并能生成质量可用的 `paper_content.txt`。

本论文集原则上不应作为重点收录以下论文：

1. 纯综述、纯教程、纯标准解读、纯方法论或纯工具论文。
2. 只讨论开发流程、验证流程、建模流程、需求工程流程，而没有控制对象行为描述的论文。
3. 只关注连续控制性能、参数调优、轨迹优化、感知估计或诊断算法，而缺少离散控制逻辑的论文。
4. 虽然出现 automata / state machine / mode 等词，但对象其实是工具流程、文档流程、软件过程或实验流程的论文。

## 4. 纳入与排除判定标准

后续筛选时，至少从以下维度判断：

1. 研究对象
   - 纳入：对象是实际控制系统、控制器或控制子系统。
   - 排除：对象是方法流程、验证流程、工具工作流、架构分层或抽象元模型。
2. 任务类型
   - 纳入：原文能支持提取控制设计、行为描述、模式切换、动作顺序或异常恢复逻辑。
   - 排除：原文只支持总结算法、框架、综述观点或验证工具能力。
3. 证据形态
   - 纳入：正文、图注、表格、变量说明中存在足够的可追溯证据。
   - 降优先级：证据大多散落在图中、图注中，需要额外整理但仍有希望形成 `STM.md`。
   - 排除：几乎没有可追溯文本，或文本质量差到无法可靠核对。
4. 可提取性
   - 纳入：可以生成 `paper_content.txt`，并据此编写 `STM.md`。
   - 排除：PDF 无法获取，或即使用 OCR 后仍不能支持可靠提取。
5. 与本研究相关性
   - 纳入：能直接为 `project_1` 的“自然语言到状态机建模”任务提供样本、案例、对象类型或控制逻辑模式。
   - 降优先级：只提供背景知识，不能直接形成状态机描述样本。

## 5. 本论文集下文件说明

本论文集默认包含以下核心文件：

1. [README.md](./README.md)
   - 入口说明文件。
   - 先回答“这个论文集是什么、为什么存在、收什么、不收什么”。
2. [GUIDE.md](./GUIDE.md)
   - AI 工作操作规范。
   - 规定检索、筛选、目录维护、[SUMMARY.md](./SUMMARY.md) 回填、失败处理和一致性检查方式。
3. [SUMMARY.md](./SUMMARY.md)
   - 当前论文集的总账。
   - 记录统计信息、关键词簇、论文清单、失败记录和 `STM.md` 收获盘点。
4. [STM_GUIDE.md](./STM_GUIDE.md)
   - `STM.md` 的专项规范。
   - 规定什么样的对象算有效控制系统样本、如何组织证据、如何写“未收获”。

AI 在开始具体工作前，推荐阅读顺序为：

1. [README.md](./README.md)
2. [GUIDE.md](./GUIDE.md)
3. [SUMMARY.md](./SUMMARY.md)
4. [STM_GUIDE.md](./STM_GUIDE.md)（仅当任务涉及 `STM.md`）
5. 目标论文目录下的 `bibtex.bib`
6. 目标论文目录下的 `paper_content.txt`
7. 必要时核对 `paper.pdf`

## 6. 单论文路径约束

本论文集下每个单论文目录默认至少应包含：

1. `paper.pdf`
2. `paper_content.txt`
3. `bibtex.bib`
4. `STM.md`

其中：

1. `paper_content.txt` 必须优先使用 `tools/pdf_extractor.py` 生成，默认先走 `text` 模式；若提取异常，立即切换 `ocr` 模式。
2. `STM.md` 是本论文集的核心派生文件，必须遵循 [STM_GUIDE.md](./STM_GUIDE.md)。
3. `desc.md` 在本论文集中不是强制文件；除非任务另有要求，不默认批量生产 `desc.md`。

## 7. AI 工作入口提示

进入本论文集时，默认应按以下方式工作：

1. 先读 [README.md](./README.md)，理解边界，避免误收。
2. 再读 [GUIDE.md](./GUIDE.md)，确认本轮工作流程。
3. 再看 [SUMMARY.md](./SUMMARY.md)，掌握当前总账、失败历史和领域分布。
4. 若需要生成或重写 `STM.md`，必须再读 [STM_GUIDE.md](./STM_GUIDE.md)。
5. 进入具体论文目录后，严格按 `bibtex.bib -> paper_content.txt -> paper.pdf（必要时） -> STM.md` 的顺序处理。
6. 完成单篇目录后，必须回写 [SUMMARY.md](./SUMMARY.md)，不能只在目录里留下未入账的半成品。

## 8. 后续 AI 应优先做什么、避免做什么

优先做的事：

1. 补齐缺失的 `paper_content.txt`、`STM.md` 和完整 `bibtex.bib`。
2. 基于已有高命中论文反推新的检索关键词簇，并回写 [SUMMARY.md](./SUMMARY.md)。
3. 优先搜索那些当前样本数偏少、但逻辑上仍容易命中有效控制对象的领域。

应避免的事：

1. 把 `sources/` 当成宽泛“形式化方法论文池”无限膨胀。
2. 只看标题或摘要就判断论文是否可用。
3. 只新增目录而不更新 [SUMMARY.md](./SUMMARY.md)。
4. 为了凑条目数，把并非控制系统运行逻辑的流程、工具状态或方法步骤写进 `STM.md`。
