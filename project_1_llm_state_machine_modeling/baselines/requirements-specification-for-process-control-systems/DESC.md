## 简报

本文关注的问题是：为复杂过程控制系统提供一种既形式化又可审阅的需求规格写法，使其能够形成可验证的系统模型。

- **输入**：过程控制系统需求
- **方法**：提出适用于过程控制的需求规格写法和语言，并在 TCAS II 上构建形式化模型。
- **输出**：可审阅的形式化状态化需求模型

```text
过程控制需求 -> 状态化规格语言 -> TCAS II 形式化模型
```

一句话结论：是控制系统需求转状态化模型的经典代表工作，直接支撑本课题对“控制需求 -> 行为模型”的问题定义。

## 研究问题与动机

### 问题背景
为复杂过程控制系统提供一种既形式化又可审阅的需求规格写法，使其能够形成可验证的系统模型。

### 核心问题
作者试图回答：在给定 过程控制系统需求 的前提下，怎样更稳定地得到 可审阅的形式化状态化需求模型，或至少把需求加工为更接近该目标的形式化中间层。

### 研究动机
核心动机是减少需求到行为设计之间的人工鸿沟，提高可追溯性、一致性以及后续验证/执行的可行性。
