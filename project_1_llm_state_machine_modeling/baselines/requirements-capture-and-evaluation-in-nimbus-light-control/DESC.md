## 简报

本文关注的问题是：把灯光控制系统的需求捕获为可执行、可评估的状态化规格，并借助 Nimbus 进行可视化和执行。

- **输入**：灯光控制系统需求描述
- **方法**：使用 RSML-e 作为状态化规格语言，在 Nimbus 中执行和可视化高层需求模型。
- **输出**：RSML-e 状态化需求模型

```text
灯光控制需求 -> RSML-e 状态规格 -> Nimbus 可视化/执行/评估
```

一句话结论：这是经典控制系统需求到状态化模型的直接案例，对本课题尤其有价值。

### 问题背景
把灯光控制系统的需求捕获为可执行、可评估的状态化规格，并借助 Nimbus 进行可视化和执行。

### 核心问题
作者试图回答：在给定 灯光控制系统需求描述 的前提下，怎样更稳定地得到 RSML-e 状态化需求模型，或至少把需求加工为更接近该目标的形式化中间层。

核心动机是减少需求到行为设计之间的人工鸿沟，提高可追溯性、一致性以及后续验证/执行的可行性。
