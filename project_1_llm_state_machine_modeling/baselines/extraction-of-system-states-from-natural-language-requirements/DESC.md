## 简报

本文关注的问题是：自动从自然语言需求中识别 system state 片段，减轻仿真与需求对齐时的人工标注工作。

- **输入**：自然语言需求文本
- **方法**：将状态抽取建模为命名实体识别任务，使用自训练 BiLSTM + CNN 模型，并提供自动与迭代半自动两种训练流程。
- **输出**：状态实体/状态短语

```text
自然语言需求 -> NER 训练/迭代标注 -> 状态实体列表
```

一句话结论：说明“先抽状态再做更高层状态机构建”是可行的中间步骤，能显著压缩人工工作量。

## 研究问题与动机

### 问题背景
自动从自然语言需求中识别 system state 片段，减轻仿真与需求对齐时的人工标注工作。

### 核心问题
作者试图回答：在给定 自然语言需求文本 的前提下，怎样更稳定地得到 状态实体/状态短语，或至少把需求加工为更接近该目标的形式化中间层。

### 研究动机
核心动机是减少需求到行为设计之间的人工鸿沟，提高可追溯性、一致性以及后续验证/执行的可行性。

## 核心方法

### 方法概述
将状态抽取建模为命名实体识别任务，使用自训练 BiLSTM + CNN 模型，并提供自动与迭代半自动两种训练流程。

### 方法要点
- 任务入口：自然语言需求文本
- 关键处理：将状态识别建模为 NER，并用自动/半自动方式训练。
- 最终产物：状态实体/状态短语（状态实体列表）
- 使用的LLM：未使用（BiLSTM-CNN）

## 实验与评估

### 数据集
- **数据/案例**：需求状态抽取语料
- **来源类型**：需求文档
- **制作方法**：人工/半自动标注约 2000 条需求
- **规模**：约 2000 条需求

### 实验设置
在约 2000 条需求上比较自动与迭代方案；人工提取需 9 小时，自动方案 15 分钟得 F1=0.51，迭代方案 100 分钟得 F1=0.62。

### 主要实验结果
说明“先抽状态再做更高层状态机构建”是可行的中间步骤，能显著压缩人工工作量。

### 方法的局限性
仅抽取 states，不生成 transitions 或完整状态机；仍需人工参与迭代标注与后续结构化。
