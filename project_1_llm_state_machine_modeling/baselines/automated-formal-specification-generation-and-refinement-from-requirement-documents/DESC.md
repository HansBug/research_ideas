## 简报

- **输入**：受控自然语言状态化 use cases
- **方法**：设计受限语言模板，并自动翻译为 CSP 形式模型
- **输出**：CSP 规范模型

```text
状态化 use case -> 受控语言模板 -> CSP 自动翻译
```

一句话结论：它不直接输出状态图，但它把 use case 文本自动转成状态化过程代数模型，是需求到行为模型的重要前身资料。

## 研究问题与动机

### 核心问题
如何减少自然语言需求文档中的歧义，并把其自动转为可验证的形式规范。

### 研究动机
让 use case 文档能够直接支持形式分析与测试生成。

## 核心方法

### 方法概述
作者提出一种带状态、输入输出、守卫和变量更新的受控自然语言 use case 模板，并自动翻译成 CSP。

### 方法要点
- 输入是状态化 use cases，而不是任意自由文本。
- 输出是可送入 FDR 的 CSP 规格。
- 面向工业案例与测试生成场景。

## 实验与评估

### 数据集
- **数据/案例**：Motorola 的 SMS/MMS industrial use case 案例
- **来源类型**：工业案例

### 主要实验结果
论文报告生成了 570 行 CSP，并成功用 FDR 检查性质。

### 方法的局限性
输出不是状态机图，而是 CSP；且依赖较强的受控用例模板。

## 与本研究的关系

### 相关性分析
- **BASELINE评估**：🟠
- **评估理由**：输入仍是需求文本工件，且方法自动化程度高，但最终输出是 CSP 而非状态机族模型。
- **与本研究的主要差异**：本研究强调状态机工件；该文强调过程代数形式规范。

### 可借鉴之处
其受控自然语言模板、守卫和变量更新建模方式，对控制系统需求规范化很有价值。
