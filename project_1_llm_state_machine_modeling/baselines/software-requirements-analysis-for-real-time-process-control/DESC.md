## 简报

本文关注的问题是：为实时过程控制软件需求规格定义语义分析准则，重点检查鲁棒性与无歧义性。

- **输入**：状态机式需求规格
- **方法**：以抽象状态机模型为基础定义分析准则，再据此对具体状态机规格语言做语义分析。
- **输出**：需求分析准则与状态机式语义检查框架

```text
状态机需求规格 -> 抽象状态机分析准则 -> 鲁棒性/歧义性检查
```

一句话结论：对本研究的价值在于提供可作为“生成后检查”的老牌语义分析口径。

## 研究问题与动机

### 问题背景
为实时过程控制软件需求规格定义语义分析准则，重点检查鲁棒性与无歧义性。

### 核心问题
作者试图回答：在给定 状态机式需求规格 的前提下，怎样更稳定地得到 需求分析准则与状态机式语义检查框架，或至少把需求加工为更接近该目标的形式化中间层。

### 研究动机
核心动机是减少需求到行为设计之间的人工鸿沟，提高可追溯性、一致性以及后续验证/执行的可行性。
