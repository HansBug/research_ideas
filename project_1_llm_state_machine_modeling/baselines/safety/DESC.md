## 简报

**解决的问题**：本文利用大语言模型（LLMs）实现安全状态图的自动扩展和测试用例生成，解决了传统方法依赖测试工程师经验、测试质量不稳定、测试效率低下的问题。

- **输入**：
  - 基本状态图（表达功能逻辑）
  - 航空安全标准（DO-178C、ARP 4754A）提取的安全准则
  - 测试路径信息

- **方法**：
  - 安全准则提取：从DO-178C和ARP 4754A中提取4类25项安全准则
  - LLM状态图扩展：使用微调的Qwen2-72B-Instruct模型扩展状态图，添加安全状态（数据验证、故障诊断、故障处理）
  - 测试用例生成：边优先深度遍历生成测试路径 + LLM优化的遗传算法生成测试数据

- **输出**：
  - 扩展的安全状态图（包含安全状态和迁移）
  - 完整的测试用例（测试路径 + 测试数据）

**实验结果**：在航空高度控制系统案例中，与经典遗传算法相比，平均执行轮数减少90%；与6种改进遗传算法相比，平均迭代次数减少77.32%，最高减少94.31%。LLM优化策略可增强其他改进遗传算法的性能。

**研究动机**：航空软件安全测试严重依赖测试工程师的历史经验，缺乏经验的人员容易导致安全分析视角不完整、状态图扩展存在遗漏或错误，测试质量不稳定。传统基于模型的安全分析（MBSA）方法需要手动扩展状态图，效率难以有效提升。大语言模型在内容生成和软件测试领域展现出强大能力，可用于自动化状态图扩展和测试用例生成。

**方法创新**：
1. 从航空标准中系统性提取安全准则，形成4类25项具体准则
2. 使用微调的LLM自动扩展状态图，添加安全状态和迁移，减少对人员经验的依赖
3. 提出边优先深度遍历算法生成测试路径，比节点优先遍历更全面
4. 使用LLM优化遗传算法的超参数设置和个体变异过程，显著减少执行轮数

**实验设计**：
- 案例系统：航空高度控制系统（8个状态：Start、InitialVerification、AltitudeMonitor、Climbing、CompleteClimb、Descending、CompleteDescding、Holding）
- 对比方法：6种遗传算法（Path Coverage GA、Self Adaptation GA、Complex System GA、Negative selection GA、Scalable Parallel GA、Multiple Paths GA）
- 对比维度：无LLM干预、LLM超参数生成、LLM个体变异
- 评估指标：平均执行轮数、算法稳定性（波动性）
- 使用模型：Qwen2-72B-Instruct（使用LoRA技术微调）
- 验证工具：UPPAAL（验证扩展状态图的完整性和一致性）

**结论与不足**：
- 主要结论：LLM可有效自动化安全状态图扩展和测试用例生成，显著提升测试效率，减少对人员经验的依赖
- 优势：执行轮数大幅减少（90%），适用于具有类似安全属性的系统，LLM优化策略可增强多种遗传算法
- 局限性：
  1. 安全准则覆盖有限（仅4类25项），未涵盖控制逻辑、处理逻辑、传输过程等方面
  2. 测试路径生成未考虑不同迁移之间的上下文关系，可能导致路径执行条件冲突或不完整
  3. 仅在单个航空案例上验证，缺乏多领域、多系统的泛化性验证
  4. 依赖大规模LLM（Qwen2-72B），计算资源需求较高
  5. 未提供与人工扩展方法的对比实验，难以量化自动化方法的准确性
