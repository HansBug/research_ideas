## 简报

本文研究如何从 LSC 这类场景化交互规格自动综合出满足要求的状态化对象系统。

- **输入**：LSC 规格
- **方法**：定义一致性与综合条件，若存在满足规格的对象系统则自动构造
- **输出**：状态化对象系统 / state-based object systems

```text
LSC 规格 -> 一致性检查与综合算法 -> 状态化对象系统
```

一句话结论：这是 scenario/sequence-specification 到 state-based system 的经典自动综合工作，对后续 statechart synthesis 系列论文影响很大。

## 研究问题与动机

### 问题背景
序列图和 use-case 场景常被拿来描述交互需求，但实现落地仍需状态化模型。

### 核心问题
给定 LSC 规格，是否存在满足该规格的对象系统；若存在，如何自动构造。

### 研究动机
让高层场景规格更直接地通向可分析、可执行的状态化系统设计。

## 核心方法

### 方法概述
论文把综合问题形式化为“判定 + 构造”问题：先判断 LSC 规格是否一致，再自动生成满足规格的状态化对象系统。

### 方法要点
- LSC 比普通 MSC 更强，能表达禁止情形与强制行为。
- 综合结果面向对象系统，而不只是单个 trace 自动机。
- 强调从场景规格直接导向可实现系统。

## 实验与评估

### 数据集
- **数据/案例**：LSC 规格示例
- **来源类型**：方法论文示例
- **规模**：以理论与算法阐述为主，未给统一 benchmark

### 主要实验结果
作者展示了在 LSC 输入充分的前提下，自动综合满足需求的状态化对象系统是可行的。

### 方法的局限性
输入前提较强，需要先把需求整理成 LSC；对自然语言和工业控制需求文本本身不直接处理。
