## 简报

本文关注的问题是：让需求工程师用受限自然语言写需求，并自动得到无歧义解释、时序逻辑和分析入口。

- **输入**：受限自然语言 FRETish 需求
- **方法**：对每条 FRETish 需求生成自然语言解释、图形解释、未来/过去时序逻辑，并支持交互式仿真。
- **输出**：FRETish 形式化需求 + 时序逻辑公式

```text
FRETish 需求 -> 英文/图形解释 -> 时序逻辑 -> 交互仿真/分析工具
```

一句话结论：FRET 是需求形式化领域的代表工具，适合作为自然语言到形式化中间层的强参照。

## 研究问题与动机

### 问题背景
让需求工程师用受限自然语言写需求，并自动得到无歧义解释、时序逻辑和分析入口。

### 核心问题
作者试图回答：在给定 受限自然语言 FRETish 需求 的前提下，怎样更稳定地得到 FRETish 形式化需求 + 时序逻辑公式，或至少把需求加工为更接近该目标的形式化中间层。

核心动机是减少需求到行为设计之间的人工鸿沟，提高可追溯性、一致性以及后续验证/执行的可行性。
